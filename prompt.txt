I'd like to change the tool now to make it easier to use.

First the tool is doing 2 things:
  - normalize yaml files (sort keys, sort arrays, ...)
  - compare yaml files (optionally ignore some differences)

For now we only run `k8s-diff` and it do both. I'd like to:
  - rename `k8s-diff` tool to `ky` (Kubernetes Yaml)
  - define command `lint` to run normalization (sort keys, sort arrays, remove fields ...)
  - define command `diff` to compare manifests.

Here are example how it should be used:

```bash
# Build kustomize manifest then normalize output using ky
kustomize build path/to/directory | ky

# Build Helm manifest then normalize output using ky
helm template  . | ky

# Lint (normalize) a Kubernetes manifest
ky lint path/to/yaml/file.yml

# Lint (normalize) Kubernetes manifests (in a directory)
ky lint path/to/directory

# Compare (diff) Kubernetes manifests
ky diff path/to/first/manifest.yaml path/to/second/manifest.yaml (other options)

# Compare (diff) Kubernetes manifests (in a directory)
ky diff path/to/first/directory path/to/second/directory (other options)
```

Let's make a plan for refactoring the tool.
